# PRD: 100% Test Coverage for sops-operator

## Overview
Achieve 100% test coverage as reported by Codecov.io for the sops-operator project. This involves adding tests for uncovered code paths in `pkg/sops/decrypt.go` and ensuring the codecov.yml configuration properly excludes non-production code.

## Goals
- Achieve 100% coverage target in Codecov.io
- Cover all edge cases in `pkg/sops/decrypt.go` (currently at 98.2%)
- Ensure codecov.yml correctly excludes generated code, test utilities, and cmd entry point
- Update CI configuration to enforce 100% coverage requirement

## Quality Gates

These commands must pass for every user story:
- `go test -race -coverprofile=coverage.out ./...` - Run tests with coverage
- `go tool cover -func=coverage.out | grep "total:" | awk '{print $3}'` - Verify coverage percentage
- `golangci-lint run` - Linting

## User Stories

### US-001: Add test for tmpFile.Close() error path
As a developer, I want to test the file close error handling in `runSopsDecrypt` so that all error paths are covered.

**Acceptance Criteria:**
- [ ] Add test that triggers `tmpFile.Close()` error at decrypt.go:194-195
- [ ] Use mock file implementation that returns error on Close()
- [ ] Verify error message contains "failed to close temp file"
- [ ] `go tool cover -func` shows 100% for `runSopsDecrypt`

### US-002: Add test for yaml.Marshal error path
As a developer, I want to test the yaml.Marshal error handling in `parseDecryptedYAML` so that all error paths are covered.

**Acceptance Criteria:**

- [ ] Use a complex type that cannot be marshaled (e.g., channel or function)
- [ ] Verify error message contains "failed to marshal value"

### US-003: Update codecov.yml for 100% target
As a project maintainer, I want to configure Codecov.io to require 100% coverage so that coverage regressions fail CI.

**Acceptance Criteria:**
- [ ] Update codecov.yml `target` from 80% to 100%
- [ ] Update codecov.yml `threshold` from 5% to 0%
- [ ] Verify ignore patterns cover all non-production code:

  - `test/**/*` (e2e test utilities)
  - `**/zz_generated*.go` (generated code)

## Functional Requirements
- FR-1: All testable production code paths must be exercised by unit tests
- FR-2: Error handling paths must be tested using mocks/stubs
- FR-3: Codecov.io configuration must enforce 100% coverage target
- FR-4: Coverage reports must exclude generated code and test utilities
